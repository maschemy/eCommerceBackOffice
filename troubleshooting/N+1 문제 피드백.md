# 🚀 Trouble Shooting: N+1 문제 해결

---

## 📌 배경 (Background)
* **상황:** 주문 리스트 조회 기능 구현 중, 고객·상품·관리자 정보를 함께 가져와야 하는 상황 발생.
* **이슈:** 단순 쿼리 메서드(`Repository` 메서드) 사용 시 **N+1 문제**가 발생한다는 튜터님의 피드백을 받았습니다.

---

## 🧐 N+1 문제란?
* **정의:** 1번의 쿼리로 N개의 데이터를 조회한 후, 연관된 데이터를 가져오기 위해 다시 **N번의 추가 쿼리**가 실행되어 성능이 급격히 저하되는 현상입니다.

---

## 💡 해결 방안: JPQL의 `JOIN FETCH`
* **설명:** 연관된 엔티티를 한 번의 쿼리로 함께 조회(Eager Loading)하여 추가 쿼리 발생을 원천적으로 방지합니다.
* **특징:** 엔티티에 설정된 `fetchType`보다 **우선순위**를 가집니다.

---

## 📝 수정 코드 (Refactoring)
주문 관리 시스템에 적용된 튜터님의 코드를 참고하여, **상품별 리뷰 조회 기능**에 `JOIN FETCH`를 적용했습니다.

```java
@Query("SELECT r FROM Review r " +
       "JOIN FETCH r.order o " +           // 주문과 조인
       "JOIN FETCH o.customer c " +        // 고객과 조인
       "WHERE r.deletedAt IS NULL " +
       "AND o.product.id = :productId " +
       "ORDER BY r.createdAt DESC " +
       "LIMIT 3")
List<Review> findLatestReviewsByProductId(@Param("productId") Long productId);
